PR B — GiselleThemeProvider — DETAIL
--------------------------------------
Already committed prop interface in
src/components/theming/theme-provider/giselle/types.ts:

  export interface GiselleThemeProviderProps {
    children: ReactNode;
    themeOverrides?: CssVarsThemeOptions;  // merged on top of Giselle defaults
    theme?: CssVarsTheme;                  // fully custom; ignores themeOverrides
    defaultMode?: 'light' | 'dark' | 'system';  // @default 'system'
  }

What the rm session must know:
- Must use extendTheme() + CssVarsProvider from MUI v7 (CSS variables mode)
  NOT createTheme() + ThemeProvider (the Minimal pattern)
- All palette refs inside giselle-mui use theme.vars.palette.* (CSS variable
  channel refs), never theme.palette.* — non-negotiable for dark mode
- Minimal's componentsProps is MUI v6 — giselle-mui uses slotProps (MUI v7+)
- giselle-mui has a CI forbidden-words check — any Minimal-derived code must
  be scrubbed of Minimal identifiers before landing here


COPYRIGHT / EXTRACTION RULES
------------------------------
Governs anything moving from rm (Minimal-derived) -> giselle-mui (open-source):

1. No Minimal utility functions — do not copy bgGradient(), textGradient(),
   alpha(), or any Minimal helper verbatim. Rewrite from MUI CSS vars primitives.
2. No Minimal config structure — CONFIG.assetsDir, CONFIG.site etc. must be
   replaced with explicit props (e.g. backgroundImageSrc?: string).
3. No Minimal component names — rename everything (GiselleThemeProvider,
   GiselleSettingsProvider, etc.).
4. All palette tokens must use theme.vars.palette.* — never #hex literals,
   never rgba(), never theme.palette.* (non-variable form).
5. channelAlpha helper — giselle-mui has its own channelAlpha(mainChannel, 0.08)
   in src/utils/theme-utils.ts. Use this instead of Minimal's alpha helper.


API DESIGN RULES — TIER SYSTEM (brief)
----------------------------------------
Every component is tiered before props are designed:

Tier 1 — Pure extension
  Extends MUI base, zero new props. Convention enforcement only.

Tier 2 — Selective extension
  Extends MUI base, adds only genuinely new props (ReactNode slots,
  narrowed color type). JSDoc only on own props, never inherited MUI props.

Tier 3 — Composition
  Data-driven (items: Item[]). Does not extend a specific MUI base.
  Item type is the real API.

GiselleThemeProvider = Tier 2 (wraps CssVarsProvider, adds themeOverrides
and defaultMode — no re-declaration of MUI props).

Full rules: docs/components/api-design-rules.md in giselle-mui.
